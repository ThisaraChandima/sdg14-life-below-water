2. Splash Screen

index.html

The Splash Screen is the site's entry scene. Per spec 4.4 it is exempt from the shared header/nav/footer template, so it has its own single, full-screen layout rather than reusing the site's flex-column body structure.

HTML Structure

  • <meta http-equiv="refresh" content="4;url=home.html"> implements the required automatic redirect to the Home Page after four seconds.
  • <main class="splash-screen"> holds the whole scene as a single semantic landmark.
  • Decorative sonar rings (.splash-sonar) and the tide-gauge loader are marked aria-hidden="true", since they carry no information of their own.
  • Content is grouped in .splash-content: an eyebrow label, an <h1> title, a tagline, a short mission statement, an unordered list of the four team members, a live countdown paragraph, and a Skip Intro button.
  • The countdown paragraph uses aria-live="polite" so its updates are announced to assistive technology as they change.

CSS Techniques

  • Links the shared style.css for colour and typography tokens (so the splash screen still feels visually continuous with the rest of the site), then adds a page-specific <style> block scoped only to splash-screen classes.
  • The background is a radial-gradient built from --bg-ocean/--bg-deep rather than an external photo, so no licensed image is required and the palette stays perfectly in sync with the rest of the site.
  • Three concentric "sonar ping" rings are animated with a single @keyframes sonarPing rule, staggered using animation-delay on each ring for a continuous pulse effect.
  • clamp() is used on the title (.splash-title) for fluid, responsive sizing without extra media queries.
  • The waterline loader (.splash-waterline-fill) is a CSS-only animation (@keyframes tideRise, 3.8s) that fills left-to-right, timed to finish just as the 4-second meta-refresh fires — satisfying the "CSS loader animation" and "not a GIF/video" requirements at once.
  • Content elements fade/slide in on a staggered sequence via a shared .fade-in class combined with inline animation-delay values, so the eyebrow, title, tagline, mission text, team list and countdown appear in order rather than all at once.
  • A @media (prefers-reduced-motion: reduce) block disables the ping, waterline and fade-in animations and jumps straight to their end states, for users who have that OS-level preference set.

Design & Implementation Rationale

  • The splash screen is exempt from the shared template, but still imports style.css purely for its colour/typography variables — this keeps the transition into the Home Page seamless without duplicating design decisions in two places.
  • The sonar-ping motif was chosen deliberately to echo the diving/depth theme used later on the AIS page (the dive gauge), so the two pages I built feel like part of one coherent idea rather than two unrelated pieces.
  • The waterline bar doubles as both the required loading indicator and a thematic device — a rising tide — reinforcing the SDG 14 message instead of using a generic spinner.
  • A JavaScript-driven countdown and Skip button (in splash.js) were added alongside the meta-refresh so users have control over the transition rather than being forced to wait, satisfying the functionality requirement while keeping the HTML meta-refresh as the guaranteed fallback redirect.

3. SDG Action Impact Simulator (AIS)

ais.html

The AIS is the site's main interactive feature: users flip switches for six ocean-friendly actions and watch a "dive gauge" descend through three depth zones as their impact score rises. Concept: a diver's control console, deliberately different from the example given in the brief (Figure 5) while still meeting every functional requirement.

HTML Structure

  • Uses the shared template unmodified — <header> with nav bar, <main id="main-content">, and the standard <footer> — with only the active-page nav class changed for this page.
  • A skip link (.skip-link) is the first focusable element, letting keyboard users bypass the header.
  • .console-layout is a two-part grid: <section class="control-panel"> containing six <article class="action-row" role="switch" aria-checked> action cards (each with an inline SVG icon carrying a <title>, a heading, description, points tag and toggle lever), and <aside class="gauge-panel" aria-live="polite"> containing the SVG dive-gauge instrument and feedback text.
  • Three hand-coded inline background SVGs (#bg-low, #bg-medium, #bg-high) sit in a fixed #depth-bg layer behind everything; JavaScript toggles which one is visible.

CSS Techniques

  • Embedded <style> block scoped to this page only, reusing the global design tokens (--bg-*, --primary, --teal, --coral, --radius, --ease) rather than introducing new colours — satisfies the "must not override or conflict with the Global CSS" requirement.
  • CSS Grid (.console-layout) creates the two-column control/gauge layout, collapsing to a single column under 820px with the gauge panel reordered above the switches (order: -1) for mobile.
  • Control and gauge panels reuse the site's glassmorphism language (backdrop-filter: blur() saturate()) already established in the nav bar, for visual consistency.
  • Each action's toggle is a custom "lever" built from a pseudo-element thumb, animated with transform: translateX() on the .selected state, giving full visual control instead of relying on a native checkbox appearance.
  • Background depth-zone SVGs cross-fade via an opacity transition and a .zone-visible class, layered at z-index: -1 behind all content.

JavaScript

  • A single vanilla-JS IIFE, no external dependencies.
  • Each action row carries a data-points attribute; the total score is the sum of points on all .selected rows.
  • getLevel() maps the total against two documented thresholds (medium ≥ 7, high ≥ 14, out of a possible 20) into three named impact levels — low (Sunlit Zone), medium (Twilight Zone), high (Reef Depths).
  • updateGauge() recalculates the SVG gauge fill height and marker position proportionally to the score, swaps the visible background SVG, updates the feedback message and zone legend, and updates an .sr-only live region so screen-reader users get the same "current depth zone" information as sighted users.
  • Click and keydown (Enter/Space) listeners are both attached to every action row, so the custom switch is fully operable by mouse, touch and keyboard — not just CSS :hover/:checked tricks, per the coursework's explicit warning against CSS-only interactivity.

Design & Implementation Rationale

  • The "diver's control console" concept was chosen specifically to avoid replicating the card-grid example in Figure 5 of the brief, while still covering every listed requirement (6+ action cards, JS-driven selection, score calculation, three impact levels, JS-controlled background changes).
  • Toggle levers and a depth-gauge instrument reinforce the ocean/diving theme rather than generic checkboxes and a plain progress bar, tying this page visually to the splash screen's sonar motif.
  • Background scenes are hand-coded inline SVG rather than photographs, so they load instantly, need no external licensing or referencing, and can be recoloured using the same design tokens as the rest of the site.
  • Impact-point thresholds are defined as named constants and explained in a code comment block, per the coursework's requirement that thresholds be "clearly stated in the code/comments."

4. Content Page

content_ST2.html

The Content Page is the site's in-depth explainer on marine plastic pollution, moving from where ocean plastic actually comes from through to concrete actions a student or local community can take. Concept: an illustrated, jump-navigable "long read" that hands off directly into the Action Impact Simulator, rather than a static wall of text with nowhere to go next.

HTML Structure

  • Uses the shared template unmodified — <header> with nav bar, <main id="main-content">, and the standard <footer> — with only the active-page nav class changed for this page.
  • A skip link (.skip-link) is the first focusable element, letting keyboard users bypass the header.
  • .content-hero opens the page with the <h1>, an intro paragraph linking through to ais.html, and a .content-jumpnav that lets readers jump straight to any of the five sections below instead of scrolling past all of them.
  • Five .content-section landmarks (#sources, #impact, #microplastics, #action, #local-action) carry scroll-margin-top so jump-nav and back-to-top links land cleanly below the sticky header rather than underneath it.
  • .content-flex pairs a .content-text block with a .content-figure hand-coded inline SVG, alternating which side the figure sits on across sections for visual variety.
  • A .content-callout mid-page links directly to ais.html, and #local-action closes with a .cards-grid of three .card elements (On Campus / Local Area / Looking Ahead).
  • Every illustrative SVG is role="img" with paired <title>/<desc> elements so it has a proper accessible name and description, rather than being announced as blank content.

CSS Techniques

  • Embedded <style> block scoped entirely to .content-* classes, reusing the global design tokens (--teal, --coral, --radius, --bg-deep) rather than touching shared selectors like .container or nav — satisfies the "must not override or conflict with the Global CSS" requirement.
  • .content-flex is a wrapping flexbox (flex-wrap: wrap, gap) so each text/figure pair collapses to a single column on narrow screens without a dedicated media query.
  • .content-jumpnav pill links reuse the teal/coral token palette and invert colour on :hover, giving a clear, low-effort affordance for the in-page navigation.
  • scroll-margin-top on .content-section stops the sticky header from covering the top of a section whenever a jump-nav or back-to-top link is followed.
  • .back-to-top is a fixed-position circular button with a hover lift (transform: translateY()), matching the interaction language already used for buttons elsewhere on the site.

Design & Implementation Rationale

  • Five inline SVG illustrations (land-based outfall into a river, a turtle near floating plastic, microplastic fragments, a hand recycling a bottle) were hand-coded rather than sourced as photographs, so nothing needed external licensing and every illustration could be recoloured with the same tokens used on the Splash Screen and AIS.
  • The jump-nav and section anchors were added specifically so this longer, text-heavier page doesn't read as an unrelated wall of text — a visitor can skip straight to the part they care about, or read start to finish into the callout that hands off to the Simulator.
  • The mid-page callout and closing cards-grid exist to turn a purely informational page into an actionable one, tying content_ST2.html directly into ais.html instead of leaving it as a dead end.

5. Challenges & Lessons Learned

Draft — personaliseThese are drafted from what the code shows was genuinely non-trivial. Edit them into your own words and add anything else you actually struggled with.

  • Converting the impact score into an SVG coordinate space for the dive gauge (using fixed GAUGE_TOP/GAUGE_BOTTOM values rather than a simple percentage) took some trial and error to get the fill and marker to move smoothly and proportionally.
  • Making the custom action switches fully keyboard-accessible meant remembering to add explicit keydown handling for Enter/Space — a native <input type="checkbox"> would have gotten this for free, but a custom role="switch" element does not.
  • Coordinating the splash screen's automatic meta-refresh with the JS-driven Skip button required checking that clicking Skip actually gets the user to home.html without a stray redirect still pending underneath.
  • Keeping page-specific embedded CSS from clashing with the Global CSS meant scoping every new rule tightly to page-specific classes/IDs (e.g. #depth-bg, .console-layout) instead of touching shared selectors like .container or nav directly.

6. Accessibility Considerations

  • A skip link on the AIS page ("Skip to main content") lets keyboard and screen-reader users bypass the repeated header/nav on every visit.
  • Purely decorative graphics — sonar rings, the depth-zone background scenes — are marked aria-hidden="true" so they are not announced; meaningful icons (e.g. the action-card SVGs) instead carry a descriptive <title> element as their accessible name.
  • The AIS action switches use role="switch" with aria-checked, are keyboard-focusable (tabindex="0"), and show a visible :focus-visible outline, in addition to working with mouse and touch.
  • Both the gauge feedback panel (aria-live="polite") and the splash screen countdown use ARIA live regions so their dynamic updates reach screen-reader users; the AIS additionally has a dedicated .sr-only text node that states the current depth zone in words.
  • A prefers-reduced-motion media query on the splash screen removes or shortens the ping, waterline and fade-in animations for users who have that OS-level preference enabled.
  • All meaningful images and icons carry descriptive alt text or an SVG <title>, per the coursework's alt-text requirement.

7. Compliance with JANET Regulations

  • All page content, copy and code on the Splash Screen and AIS pages is original work written for this coursework; no text is copied from the UN SDG website or elsewhere.
  • No unlicensed copyrighted material is published — background scenes and icons are hand-coded SVG rather than sourced images, so no external licensing was needed for these two pages.
  • The site contains no illegal, defamatory, offensive or misleading material and is used strictly for the educational purpose of this coursework, in line with JANET's acceptable-use policy.
  • The footer on every page links directly to the W3C accessibility guidelines and the JISC/JANET Acceptable Use Policy, making the site's compliance posture visible to any visitor.
  • Neither the Splash Screen nor the AIS collects, stores or transmits any personal data about real third parties — the AIS state exists only in the browser session and is never sent to a server.
